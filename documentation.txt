2. Merging ERA5 files
- since there are limits to how much ERA5 data you can download at once, I often have to merge the files manually into one big dataset
- I use the package xarray to deal with this type of data
- I usually download ERA5 by year as grib files, and then merge the years into one netcdf file
(grib has less errors with ERA5 downloading. Netcdf is way more flexible on what you can do with it so that's why I convert.)
- the script merge-era5.py finds all the grib files with a given naming convention and merges them by year into one netcdf file
- file size: the 30-year files are too big to handle at once. splitting into longitude strips allows us more
maneuverability. That way we can easily calculate the 30-year mean at each grid point because the years are lumped together,
things are juts separated into chunks spatially. Each grid point has its own 30-year mean.

3. Selecting the domain
- the region for ARCC is not rectangular, which makes selecting the domain a bit more complicated
- use regionmask package https://regionmask.readthedocs.io/en/stable/ which can sort grid points into geospatial regions
- inuit regions from https://osdp-psdo.canada.ca/dp/en/search/metadata/NRCAN-FGP-1-f242b881-75e3-40bb-a148-63410b4ce2af

Extra Documentation:
1. Downloading ERA5 data yourself
- make a CDS account if you don't already have one (https://cds.climate.copernicus.eu)
The data can be downloaded directly from the website (
 https://cds.climate.copernicus.eu/datasets/reanalysis-era5-single-levels?tab=download).
However, for large amounts of data it can be tedious as there are limits to the amount you can request at a time.

 API option:
- follow the instructions to download the api (https://cds.climate.copernicus.eu/how-to-api)
- make a .cdsapirc file and set up url and key (instructions at same link above)
- python code for api requests can be found at the bottom of the download webpages (linked above)
- can add loops to the code to batch download large amounts of data (see my script api-request_old.py)
